# API Coverage Matrix

This document maps meaningful UniFi resource concepts across two client surfaces:
the **Legacy Internal API** (`unifi.InternalClient`) and the **Official OpenAPI**
(`unifi/official.Client`). It is generated by following the procedure in
`.claude/skills/unifi-coverage-matrix/SKILL.md` and reflects the live accessor
methods in `unifi/official/client.generated.go` and the resource types in
`unifi/*.generated.go` at the time of last regeneration.

## Legend

| Mark | Meaning |
|------|---------|
| ✅   | Covered — full CRUD or the natural scope of this resource is accessible |
| ⚠️   | Partially covered — some operations exist but the surface is incomplete |
| ❌   | Not covered — no equivalent on this surface |
